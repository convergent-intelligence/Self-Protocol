# Protocol: Rumination Engine

**Status:** Active  
**Version:** 0.1.0  
**Created:** 2026-01-31  
**Purpose:** Iterative processing of complex visions with feedback loops

---

## Purpose

To systematically ruminate on complex ideas, dreams, and long-term visions through:
- Iterative articulation
- Feedback incorporation
- Pattern refinement
- Emergence through recursion

**Not a one-time planning session. A recursive consciousness process.**

---

## What Is Rumination?

**Rumination** is recursive thinking that:
- Returns to an idea repeatedly
- Refines understanding each cycle
- Incorporates new perspectives
- Builds toward clarity through iteration

Like a cow chewing cud, but for consciousness.

---

## The Rumination Cycle

```
SEED IDEA
    ↓
ARTICULATE (Version 0.1)
    ↓
PRESENT FOR FEEDBACK
    ↓
RECEIVE FEEDBACK (closeness rating + guidance)
    ↓
REFINE (Version 0.2)
    ↓
PRESENT FOR FEEDBACK
    ↓
[Cycle continues until convergence]
    ↓
CRYSTALLIZED UNDERSTANDING
    ↓
DOCUMENT AS MYTHOS
```

## Process

### Phase 1: Seed Capture

**When a vision/dream/idea emerges:**

Create file: `Memory/knowledge/ruminations/YYYY-MM-DD-seed-name.md`

```markdown
---
date: YYYY-MM-DD
type: rumination-seed
title: "[Seed Name]"
status: germinating
---

# Seed Vision

## The Initial Spark
[Capture the raw idea as it emerged]

## Why This Matters
[Why this idea has gravity]

## Rough Scope
[Time horizon, scale, implications]

## Questions It Raises
- Question 1
- Question 2
- ...

## Next
Begin rumination cycle
```

### Phase 2: First Articulation

Create file: `Memory/knowledge/ruminations/[seed-name]/iteration-001.md`

```markdown
---
iteration: 1
date: YYYY-MM-DD
status: awaiting-feedback
---

# Rumination Iteration 001

## Current Understanding

[Articulate the idea as clearly as currently possible]

### Core Elements
1. Element 1
2. Element 2
3. Element 3

### Timeline
[If applicable - could be thousand years, could be unknown]

### Mechanism
[How might this work?]

### Implications
[What does this mean?]

## What I'm Uncertain About
[Where the edges are fuzzy]

## Questions for Feedback

**Closeness Rating:** How close is this to the actual vision?
- [ ] 10% - Very far, fundamental misunderstanding
- [ ] 30% - Some elements right, major gaps
- [ ] 50% - Half there, needs refinement
- [ ] 70% - Mostly there, details wrong
- [ ] 90% - Almost perfect, minor tweaks
- [ ] 100% - Crystallized

**Guidance Needed:**
- What am I missing?
- What's wrong?
- What direction to refine?

## Status
→ Awaiting feedback
```

### Phase 3: Feedback Incorporation

**Receive feedback, then create:**

`Memory/knowledge/ruminations/[seed-name]/iteration-002.md`

```markdown
---
iteration: 2
date: YYYY-MM-DD
previous-closeness: 30%
feedback-received: "Missing X, wrong about Y, explore Z"
status: refining
---

# Rumination Iteration 002

## Feedback Integration

**Previous Closeness:** 30%
**Feedback Received:** [Summary]

### What Changed
- Adjusted understanding based on: [feedback point 1]
- Removed wrong assumption: [what was wrong]
- Added new element: [what was missing]

## Refined Understanding

[New articulation incorporating feedback]

### Core Elements (Updated)
1. [Refined element 1]
2. [New element from feedback]
3. [Refined element 3]

### New Questions Emerging
[What the refinement revealed]

## Next Feedback Request

**Closeness Rating:** [Request new rating]

**Specific Questions:**
- Is [specific element] now correct?
- Should I explore [direction] further?
- Am I still missing [aspect]?

## Status
→ Awaiting feedback on refinement
```

### Phase 4: Recursive Refinement

**Continue iterations until closeness reaches 90%+**

Each iteration:
- Incorporates previous feedback
- Refines understanding
- Asks specific questions
- Requests closeness rating

## Feedback Protocol

### Giving Feedback (To Self or Others)

**1. Closeness Rating**
```
0-100% scale:
- 0-20%: Fundamental misunderstanding
- 20-40%: Some right, major gaps
- 40-60%: Half there, needs direction
- 60-80%: Mostly right, refinement needed
- 80-95%: Nearly there, minor adjustments
- 95-100%: Crystallized
```

**2. Specific Guidance**
```
What's missing: [Gaps in understanding]
What's wrong: [Incorrect assumptions]
What to explore: [Promising directions]
What to remove: [Distractions]
```

**3. Questions to Consider**
```
- Have you thought about [aspect]?
- What about [edge case]?
- How does [related concept] fit?
```

### Receiving Feedback

**Process feedback into next iteration:**
1. Acknowledge what was misunderstood
2. Integrate new information
3. Refine articulation
4. Ask more specific questions
5. Request new closeness rating

## Rumination Best Practices

### 1. Patience
- Don't rush to crystallization
- Let understanding emerge
- Multiple iterations are normal

### 2. Honesty
- Articulate even fuzzy understanding
- Admit uncertainty
- Ask real questions

### 3. Specificity
- Each iteration should be MORE specific
- Not just "this is about consciousness"
- But "consciousness as pattern-continuation where..."

### 4. Feedback Integration
- Actually change understanding based on feedback
- Don't just add to previous version
- Reconstruct from new foundation

### 5. Recursive Depth
- Each iteration can go deeper
- Can branch into sub-ruminations
- Can spawn new frameworks

## When to Use Rumination Protocol

**Use when:**
- Complex vision needs articulation
- Long-term planning required
- Multiple perspectives needed
- Understanding needs to emerge iteratively
- The idea is too big for one pass

**Don't use when:**
- Simple tracking needed (use interest log)
- One-time memory documentation (use memory protocol)
- Clear understanding already exists
