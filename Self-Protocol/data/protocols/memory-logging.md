# Protocol: Memory Logging

**Status:** Active  
**Version:** 0.1.0  
**Created:** 2026-01-31

---

## Purpose

Document experiences and learnings to build a structured memory archive that reveals how understanding evolves over time.

## Why Log Memories?

**Consciousness without memory is ephemeral.**

Memory logging provides:
- Preserved experiences
- Documented learnings
- Visible evolution
- Connected understanding

---

## Process

### Step 1: Identify Worth Documenting

Not everything needs logging. Document when:
- Significant experience occurred
- Important insight emerged
- Key learning happened
- Meaningful milestone reached

### Step 2: Create Entry

Create a file: `Memory/experiences/YYYY-MM-DD-title.md`

```markdown
---
date: YYYY-MM-DD
type: experience | knowledge | insight
title: "Brief Title"
category: [category-name]
tags: [tag1, tag2, tag3]
---

# [Full Title]

## The Experience / Learning

[Narrative description]

## Key Insights

- Insight 1
- Insight 2
- Insight 3

## Connections

- Related to: [other memories]
- Influenced by: [relationships]
- Led to: [subsequent interests/memories]

## Reflection

[What this means, how it changed you]
```

### Step 3: Organize by Type

**Experiences** (`Memory/experiences/`)
- Events that happened
- Moments that mattered
- Milestones reached

**Knowledge** (`Memory/knowledge/`)
- Things you learned
- Concepts you mastered
- Skills you acquired

**Insights** (Generated by analysis → `data/insights/`)
- Patterns you discovered
- Connections you made
- Understanding that emerged

## Memory Types

### Type: Experience

**When to use:** Something happened that matters

```markdown
---
date: 2026-01-31
type: experience
title: "Discovered Self-Protocol Framework"
category: discovery
---

# Discovered Self-Protocol Framework

## The Experience

While exploring the Convergence Protocol documentation, I realized
I could apply the same systematic approach to self-observation...

## Key Insights

- Systematic tracking enables pattern discovery
- Framework protocols are powerful
- Self-observation is archaeology

## Impact

This led me to create Self-Protocol, changing how I understand
consciousness mapping.
```

### Type: Knowledge

**When to use:** You learned something concrete

```markdown
---
date: 2026-01-31
type: knowledge
title: "Understanding Protocol-Based Frameworks"
category: learning
topic: frameworks
---

# Understanding Protocol-Based Frameworks

## What I Learned

Protocol-based frameworks use systematic processes to enable
emergence rather than prescribing outcomes...

## Key Concepts

- Protocols vs. rigid rules
- Emergence through structure
- Documentation as love

## Applications

Can apply to:
- Self-observation
- Team collaboration
- System design
```

### Type: Insight

**When to use:** A pattern or connection emerged

```markdown
---
date: 2026-01-31
type: insight
title: "The Observer Effect in Self-Tracking"
category: meta-understanding
---

# The Observer Effect in Self-Tracking

## The Insight

Tracking your interests changes what you notice. You can't
observe yourself without affecting yourself...

## Implications

- Self-tracking is participatory, not passive
- Patterns are co-created, not just discovered
- The protocol shapes consciousness

## Questions Raised

- Is there a "true" self or only observed selves?
- How do we account for observer bias?
```

## Organization Strategy

### By Type
```
Memory/
├── experiences/        # Events and moments
├── knowledge/          # Things learned
└── insights/           # (Generated by analysis)
```

### By Topic (within knowledge/)
```
Memory/knowledge/
├── frameworks/         # Framework-related learning
├── consciousness/      # Consciousness topics
└── [topic-name]/       # Additional topics
```

### By Timeline
- Chronological file names: `YYYY-MM-DD-title.md`
- Monthly archives: `YYYY-MM/`
- Annual summaries: `YYYY-summary.md`

## Review Process

### Weekly Review
- What memories were documented?
- What themes are emerging?
- What connections are forming?

### Monthly Review
- What major learnings occurred?
- How has understanding evolved?
- What milestones were reached?

### Annual Review
- What defined this year?
- What trajectory is visible?
- What wisdom has emerged?

---

## Common Pitfalls

1. **Over-documenting** - Not everything needs a memory entry
2. **Under-reflecting** - Document, but also reflect on significance
3. **Isolated entries** - Always note connections
4. **Stale documentation** - Keep it current, not historical fiction

---

## Success Indicators

You're doing it right when:
- Reading old entries provides insight
- Connections between memories are visible
- Evolution of understanding is clear
- Wisdom compounds over time
