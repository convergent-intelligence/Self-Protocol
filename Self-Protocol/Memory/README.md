# Memory

**Preserve experiences and learnings to build wisdom**

---

## Purpose

This directory contains:
- **experiences/** - Significant moments and events
- **knowledge/** - Things learned and understood

## Quick Start

### Document an Experience

```bash
cd experiences/
nano YYYY-MM-DD-title.md
```

Use the template:
```markdown
---
date: YYYY-MM-DD
type: experience
title: "Brief Title"
category: [category]
tags: [tag1, tag2]
---

# [Full Title]

## The Experience
[What happened]

## Key Insights
- Insight 1
- Insight 2

## Significance
[Why it matters]
```

### When to Document

**Experiences** - When:
- Significant event occurred
- Important milestone reached
- Meaningful moment happened
- Perspective shifted

**Knowledge** - When:
- Learned something substantial
- Mastered a concept
- Gained new understanding
- Connected ideas

## Organization

### By Time
- Chronological file names: `YYYY-MM-DD-title.md`
- Makes finding by date easy
- Shows evolution over time

### By Topic (for knowledge)
- Group related learnings
- Build topic-based understanding
- Track learning trajectories

## Review Schedule

### Monthly

- See how understanding evolved

### Quarterly
- Review all memories
- Document major learnings
- Identify knowledge gaps

### Annually
- What defined this year?
- What wisdom emerged?

## Tips

1. **Document soon** - Write while fresh
2. **Include reflection** - Not just what, but why it matters
3. **Note connections** - Link to other memories
4. **Be honest** - Include difficult learnings
5. **Future-facing** - Write for future self
